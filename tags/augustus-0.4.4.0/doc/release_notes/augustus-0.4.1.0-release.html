1. Overview

The previous release, 0.4.0.a, made Augustus compliant with PMML 4. It also intoduced Missing Value Replacement as a feature.

This release focus of this release was to make more documentation available, covering both how to use Augustus and challenges that are faced when modeling.

Also, we have begun adding code and examples to handle Regular Segments. This allows users of Augustus to segment the data, without explicitly declaring each segment. This is 1) a slower way to process the data but also 2) a powerful modeling tool.

See the documentation for Augustus Version 0.4 for more information.

2. Point release additions

  • Regular Segmentation
  • Additional non-Gaussian distributions for CUSUM and GLR tests
  • Consumer commandline overrides for port and input file
  • Changed how the Python interrupter is envoke and removed the restriction against Python 2.5
  • Added a metadata logging framework to record statistical and operational information about the data and the model
  • Added a daemon mode. While running in this mode, configurations can be refreshed.
  • Deployment structure can be specified in the configuration to declare pre- and post-processing steps.
