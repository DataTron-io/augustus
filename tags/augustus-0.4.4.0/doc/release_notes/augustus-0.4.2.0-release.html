1. Overview

As with all releases since 0.4, this version of Augustus is compliant with PMML 4. This release adds new models and improves on ways of getting data into Augustus from Standard I/O. It also disables the optional garbage collection check for circular references in the consumer.

See the Augustus v 0.4 documentation for more information.

Top


2. Point release additions

  • Added a new model consumer, RuleSet. See the PMML spec at the Data Mining Groups's website for details about the model specification.
  • Disabled the optional garbage collection in the consumer. This was done for performance reasons. See the writeup on the Google Code Project site for details. Note: we have not disabled it on the various Producers, because those usually do not have the time constraints when running as the consumers an we have not tested all of them and verified that no circular references or stuctures exist.
  • Improved how data can be read into Augustus.
  • Continued work on the collection of metadata at various spots in the code
