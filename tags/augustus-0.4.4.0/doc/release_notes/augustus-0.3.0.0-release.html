2. Point release additions


Switch to Python v2.5

Augustus is now based on Python v2.5 throughout.


Switch to NumPy package.

Previous versions of Augustus used the Numarray package to handle a variety of mathematical and vectorized processing. Numarray has been superceded by the NumPy package. One advantage of using NumPy is that, on platform which support large memory addresses, Unitable columns of more than 2 Gb are now supported. This change has been accompanied by the change to compliance with Python v2.5.


Segment Validation for Baseline Models using ddist test.

Baseline models scored with a ddist test may be created with the producer which omit segments which do not have sufficient data as specified by the user.


Updates to Baseline Models using ddist test.

Existing Baseline models which use a ddist test may be updated with new data without having to re-process previous data used to create the original model.


Naive Bayes Model

Augustus now supports a Naive Bayes model.

3. Bugs fixed


Producing Baseline Models supporting the CUSUM test.

A bug in the producer for Baseline models did not correctly handle specification of the alternate model required for this test. This bug is now fixed.


Hidden files in directory specified to producer as data source.

Users have the option of specifying a directory where data appropriate for building models is located. Previous versions failed when the directory contained irregular files, most notably 'hidden' directories such as those beginning with a period ('.'). The current version ignores both directories and these files.
