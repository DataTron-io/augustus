2. Point release additions


Introducing user defined field types for csv reads

Unitable previously read in CSV files and associated a variable type with each column which minimizes the memory storage alloted. Thus, a CSV file in which all data can be interpreted as an integer would be associated with a unitable column with integer type. There are some instances (ex. when reading in a column from a fixed width format where the column values are usually of the form "01") where it is preferable to force a column to be a string type.

This new feature allows the user to specify the types of the columns in a CSV file. This is useful when the baseline consumer reads data from a csv file into a unitable; similarly for the baseline producer and any other function in Augustus which uses anyReader.

3. Bugs fixed


Corrected producer using local transforms

Previously, scoring raw data in batch mode required that the raw data be wrapped in a local transform. An error was induced under following circumstances:

  • Local Transformation essentially renames raw field A as B.
  • Global Transformation exists, but renames raw field A as C.
  • If a global transformation references new field with same name as local transformation, no error is incurred.

This bug affected anyone who tried to use a local transformation without batch scoring. This is now fixed.


Corrected MapValues to work with non-string input

pmmlElements explicitly casts the values from a InlineTable row to str. This str value is then used for a dictionary key. Any input to the MapValues that was not a string would fail the dictionary lookup. This bug is now fixed.


Corrected heterogeneous statistics from crashing consumer on output from baseline models with different test statistics

When a pmml file includes TestDistributions with two different test statistics, the output structure can be appropriate for one but possibly not for the other. When this happens, the consumer would crash.

The consumer will now cease to crash if you ask for an ancillary value which the distribution does not support. Adding ancillary values for models other than dDist may be added in a future release.
